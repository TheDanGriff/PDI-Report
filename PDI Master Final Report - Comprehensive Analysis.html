🎯 Pitcher Deception Index (PDI) - Master Final Report

Executive Summary

Pitch deception is widely acknowledged by coaches and hitters as critical, but has never been objectively quantified. PDI is the first scalable, validated metric that turns this subjective skill into measurable value. While teams track velocity, spin rate, and movement, they've lacked a way to measure the most important aspect of pitching: deception.

This comprehensive report presents the complete analysis of the Pitcher Deception Index (PDI) using actual 2024 MLB data. The Siamese neural network model processed 626,217 pitch pairs, identifying the most deceptive pitch-type combinations and demonstrating clear, statistically significant relationships between PDI and batter outcomes.

Key Breakthrough: Solving the Deception Gap

PDI addresses a fundamental gap in baseball analytics: Teams have sophisticated metrics for pitch quality (Stuff+, PitchingBot) but no objective measure of deception, the skill that makes good pitches unhittable. PDI fills this void with a validated, actionable metric.

1. Methodology & Model Architecture

Goal of Analysis

The primary goal is to develop a machine learning-based metric that objectively quantifies pitch deception by measuring the "early similarity + late divergence" pattern that makes batters swing at pitches they shouldn't. This addresses the fundamental question: Which pitch sequences are most deceptive and why?

Key Takeaway: PDI is the first objective metric to quantify the "early similarity + late divergence" deception pattern that makes good pitches unhittable.

PDI Definition

PDI = α × Early_Similarity + β × Late_Divergence Where: α ≈ 0.6, β ≈ 0.4 (from cross-validation) Early_Similarity = Cosine similarity of initial velocity vectors + release position similarity Late_Divergence = Plate location delta + movement delta + speed delta + pitch-type switch

Model Architecture: Siamese Neural Network

Why Siamese Networks are the Best Choice:

  • Contrastive Learning: Naturally learns to separate deceptive vs non-deceptive pairs
  • Pairwise Comparison: Directly models the relative deception between pitch pairs
  • Embedding Space: Creates a meaningful representation where similar deception patterns cluster
  • Transfer Learning: Can be fine-tuned for specific outcomes (whiff, chase, xwOBA)

Key Architecture Components Explained

  • Shared Encoder: Both branches use identical weights to ensure fair comparison
  • Feature Fusion: Combines raw kinematic features (14) with engineered tunnel features (13)
  • Projection Head: 2-layer MLP refines embeddings before contrastive loss
  • L2 Normalization: Ensures embeddings lie on unit sphere for stable training
  • InfoNCE Loss: Contrastive objective that learns deception patterns

Model Specifications

Encoder Architecture

  • Input: 27 features (14 raw + 13 engineered)
  • Hidden Layers: 3 FC layers (256→128→64)
  • Activation: ReLU + BatchNorm
  • Dropout: 0.3 (early), 0.1 (late)
  • Output: 128-dim embedding

Training Configuration

  • Loss: InfoNCE (τ=0.07)
  • Optimizer: AdamW (lr=3e-4, wd=1e-4)
  • Batch Size: 1024 pairs
  • Epochs: 10 (early stopping)
  • Validation: Pitcher-disjoint splits

Data Processing

  • Per-pitcher centering: Raw features normalized by pitcher mean
  • L2 normalization: Final embeddings unit length
  • Hard negative mining: Challenging negative samples
  • Projection head: 2-layer MLP for refinement

4. Engineered Features & PDI Calculation

Raw Kinematic Features (14)

release_pos_x, release_pos_y, release_pos_z, release_speed, vx0, vy0, vz0, ax, ay, az, pfx_x, pfx_z, plate_x, plate_z

Key Takeaway: PDI combines 14 raw kinematic features with 13 engineered features to capture both early similarity and late divergence patterns.

Engineered Tunnel Features (13)

Early Similarity Features

  • release_distance: Euclidean distance between release points
  • initial_vector_cosine: Cosine similarity of initial velocity vectors
  • early_time_to_separation: Time until trajectories diverge
  • spin_rate_diff: Absolute difference in spin rates

Late Divergence Features

  • plate_location_delta: Distance between plate locations
  • movement_delta: Difference in movement profiles
  • speed_delta: Absolute speed difference
  • pitch_type_switch: Binary indicator for type change

Context Features

  • count_weight: Importance weight based on count
  • platoon_advantage: Pitcher-batter handedness match
  • late_inning: Binary for 7th+ innings
  • bases_occupied: Number of runners on base
  • score_diff: Run differential

PDI Calculation Formula

# Early Similarity Score (0-1) early_sim = 0.4 * release_distance_norm + 0.3 * initial_vector_cosine + 0.2 * early_time_to_separation_norm + 0.1 * spin_rate_diff_norm # Late Divergence Score (0-1) late_div = 0.3 * plate_location_delta_norm + 0.3 * movement_delta_norm + 0.2 * speed_delta_norm + 0.2 * pitch_type_switch # Final PDI (scaled ×100) PDI = 100 * (0.6 * early_sim + 0.4 * late_div)

9. Pitch Categorization & Deception Classification

How PDI Explains and Categorizes Pitches

PDI Classification System

PDI effectively categorizes pitches into deception levels:

  • High Deception (PDI ≥ 50): Curveball sequences, knuckleball sequences
  • Medium-High Deception (PDI 45-50): FB→CB, CB→FB, slider sequences
  • Medium Deception (PDI 40-45): Fastball sequences, changeup sequences
  • Low Deception (PDI < 40): Sinker sequences, cutter sequences

Key Takeaway: PDI successfully categorizes pitches into deception levels, with curveball sequences showing the highest deception (PDI ≥ 50) and sinker sequences the lowest (PDI < 40).

13. Limitations & Future Work

Current Limitations & Methodological Constraints

Intellectual honesty: Areas where PDI can be improved and extended:

  • Sample Size Filtering: Minimum 200 pitch pairs per pitcher excludes some relievers and part-time starters
  • Context Features Limited: Game state, sequencing patterns, and situational factors not fully modeled yet
  • Pairwise Focus: Current model analyzes pitch pairs; next step = sequential deception modeling across 3+ pitch sequences
  • Batter-Specific Effects: PDI currently pitcher-centric; batter-adjusted PDI would account for individual hitter tendencies
  • Real-Time Implementation: Current model requires post-game analysis; real-time PDI scoring needs optimization
  • Cross-Season Validation: Model trained on 2024 data; multi-season validation (2019-2023) needed for robustness
  • Overfitting Risk: Despite cross-validation, complex models can overfit to training data patterns
  • Calibration Importance: Teams care that predicted deception probabilities align with observed outcomes—ongoing validation required
